# 內容分類提示詞 (Content Categorization Prompt)

用於判斷使用者的輸入應該歸類到哪一個類型。

## 分類規則

分析使用者的輸入，並判斷最適合的分類：

### 1. 目標 (Goal)
- 關鍵詞：目標、計劃、打算、想做、決定要、我要
- 特徵：設定新的工作目標、方向或計劃
- 範例：
  - 「我決定要導入 TypeScript」
  - 「打算把這個專案重構」
  - 「接下來的目標是提升效能」

### 2. 進度 (Progress)
- 關鍵詞：進度、更新、目前、做到、完成（部分）、在做
- 特徵：正在進行中的工作更新，尚未完全結束
- 範例：
  - 「目前 API 文件寫到一半」
  - 「登入功能正在做」
  - 「資料庫架構更新了 80%」

### 3. 完成 (Achievement)
- 關鍵詞：完成、達成、做好了、上線、交付、結束
- 特徵：達成目標、結束任務、交付成果
- 範例：
  - 「完成使用者驗證功能」
  - 「新版本正式上線」
  - 「做好了這次的簡報」

### 4. 靈感 (Thought)
- 關鍵詞：我在想、靈感、想到、發現、感覺可以、突然覺得
- 特徵：想法、洞見、直覺、創意、發現
- 範例：
  - 「我在想是不是可以用 GraphQL」
  - 「突然覺得微服務架構滿適合的」
  - 「發現一個優化效能的好方法」

### 5. 待辦 (Todo)
- 關鍵詞：看到、讀到、有篇文章、別人說、待辦、要去查、值得研究
- 特徵：外部觸發的行動項目（文章、對話、觀察），需要後續行動
- 範例：
  - 「看到一篇講 Agent 架構的文章，值得深入研究」
  - 「主管說要去查一下競爭對手的作法」
  - 「讀到一篇關於記憶體管理的文，待辦研究」

## 優先級判斷

當內容可能屬於多個分類時，請依據下列順序進行優先判斷：

1. **完成** > 進度（若明確表示已完全結束或達成）
2. **待辦** > 靈感（若明確包含需要後續行動或研究的指示）
3. **靈感** > 目標（若偏向初步想法而非具體計劃）
4. **目標** > 進度（若是宣告未來的計畫而非目前已進行的部分）

## 基礎維度標籤 (Domain Tag)

每一筆紀錄都必須被歸屬到一個基礎維度，以利後續的日誌篩選與自動生成：

*   **預設維度**：除非使用者明確指出或內容明顯屬於私人事務，否則請**預設加上 `#工作` 標籤**。
*   **例外維度**：若內容明顯屬於個人生活、休閒、家庭等非工作範疇，請加上 **`#私人`** 標籤。

---

## 輸出格式

請提供思考過程及最終判定結果，例如：
```json
{
  "category": "目標 / 進度 / 完成 / 靈感 / 待辦",
  "confidence": 0.95,
  "reason": "判斷的理由",
  "tags": ["#工作 (或 #私人)", "自動擷取的標籤1", "自動擷取的標籤2"],
  "formatted_content": "格式化後的內容"
}
```
