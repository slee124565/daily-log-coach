# Agent Skill 入門課程表
## 目標對象：行政人員（種子同仁）
## 學習目標：能獨立安裝並日常使用既有的 Agent Skill

---

## 課程設計原則

- **不教開發**：只學使用，降低心理門檻
- **動手優先**：每節課都有可完成的操作目標
- **即時有感**：每個階段結束後，學員都能感受到「這對我的工作有用」
- **漸進解釋**：先會用，再理解為什麼

---

## 課程總覽（4 個階段，共 5 次 session）

| 階段 | 主題 | 形式 | 時長 |
|------|------|------|------|
| Stage 1 | 為什麼要學這個 | 講解討論 | 30 分鐘 |
| Stage 2 | 安裝 Gemini CLI | 實機操作 | 45 分鐘 |
| Stage 3 | 安裝第一個 Skill | 實機操作 | 45 分鐘 |
| Stage 4a | 日常使用練習 | 實機 + 討論 | 60 分鐘 |
| Stage 4b | 個人化調適與回顧 | 討論 | 30 分鐘 |

---

## Stage 1：為什麼要學這個（30 分鐘）

**目標**：建立動機，消除「這很難」的恐懼

**內容**：

1. **問一個問題開場**（5 分鐘）
   > 「你現在用 AI 最大的不方便是什麼？」
   讓學員說出痛點（每次都要重新解釋、不記得上次做了什麼、不知道怎麼問）

2. **介紹「AI 主人 vs AI 用戶」**（10 分鐘）
   - 用戶：每次都從零開始教 AI
   - 主人：AI 記住你的習慣，直接開始工作
   - Skill 就是讓你變成主人的工具

3. **示範一個 Skill 的效果**（10 分鐘）
   - 直接現場演示一個 Skill 被觸發、執行的完整過程
   - 讓學員看到：一句話就能啟動複雜的工作流程

4. **說明今天之後你能做什麼**（5 分鐘）
   - 安裝好後，說「幫我記錄目標」就能自動寫日誌
   - 說「整理今天的工作」就能自動彙整進度
   - 這些都不需要學任何程式語言

**學員離開時的感受**：「這跟我有關，我想要這個。」

---

## Stage 2：安裝 Gemini CLI（45 分鐘）

**前置條件**：公司指派的 Google AI Studio 帳號（由管理員提供）

**目標**：學員能獨立在自己的電腦上啟動 Gemini CLI

> 詳細圖文說明請參閱：`docs/INSTALL.md`

**操作步驟**（逐步帶著做）：

1. 確認並安裝 Homebrew（5 分鐘）
   ```bash
   brew --version
   ```
   - 看到版本號 → 跳過安裝；看到 `command not found` → 執行安裝指令
   - 常見問題：安裝過程需輸入電腦密碼（畫面不顯示字元，屬正常）

2. 確認並安裝 Node.js（5 分鐘）
   ```bash
   node --version
   ```
   - 需要 v20 以上；若未安裝，執行 `brew install node`

3. 安裝 Gemini CLI（10 分鐘）
   ```bash
   npm install -g @google/gemini-cli
   ```
   - 逐一處理每個人的安裝問題
   - 常見問題備案：權限不足時，先確認 Node.js 透過 Homebrew 安裝，再執行 `brew reinstall node`

4. 取得並設定 API Key（15 分鐘）
   - 使用公司帳號登入 Google AI Studio 取得 API Key
   - 將以下這行加到 `~/.zshrc` 末尾：
     ```
     export GEMINI_API_KEY="你的API金鑰"
     ```
   - 執行 `source ~/.zshrc` 讓設定生效
   - 執行 `echo $GEMINI_API_KEY` 確認設定成功

5. 第一次啟動與自由練習（10 分鐘）
   - 輸入 `gemini` 啟動
   - 說：「你好，請用繁體中文自我介紹」確認正常運作
   - 讓學員自由對話 2-3 句，感受基本互動

**學員離開時的感受**：「我裝好了，而且我能用它說話。」

---

## Stage 3：安裝第一個 Skill（45 分鐘）

**目標**：學員能下載、安裝並成功觸發一個 Skill

**使用的 Skill**：`daily-log-coach`（記錄日常工作進度）

理由：對行政人員最直接有感，觸發詞語自然（「記錄目標」「完成了」「我有個想法」）

**操作步驟**：

1. **理解 Skill 的結構**（5 分鐘）
   - Skill 是一個資料夾，裡面有一個 SKILL.md
   - 不需要讀懂內容，只需要知道「放對地方就能用」

2. **取得 Skill**（10 分鐘）
   - 從指定位置下載（或由講師直接傳送壓縮檔）

3. **安裝 Skill 到正確位置**（15 分鐘）
   ```bash
   mkdir -p ~/.gemini/skills
   cp -r /path/to/daily-log-coach ~/.gemini/skills/
   ```
   - 帶著每個人確認目錄結構正確（管理員提供正確來源路徑）

4. **觸發測試**（15 分鐘）
   - 啟動 Gemini CLI：輸入 `gemini`
   - 輸入 `/skills list` 確認 Skill 已載入
   - 說：「我今天完成了 XXX」
   - 確認 Skill 自動觸發並寫入日誌

**學員離開時的感受**：「我裝了一個 Skill，它真的有在工作。」

---

## Stage 4a：日常使用練習（60 分鐘）

**目標**：學員能在真實工作情境中主動使用 Skill

**時機**：Stage 3 結束後間隔 3-5 天（讓學員有時間自己嘗試）

**開場回顧**（10 分鐘）
- 每人說一件「這幾天我用了 Skill，發生了什麼事」
- 收集卡關點

**情境練習**（30 分鐘）

分三輪，每輪給一個真實情境：

| 輪次 | 情境 | 練習目標 |
|------|------|----------|
| 第一輪 | 「早上開始工作，我想記錄今天的目標」 | 主動觸發 Skill |
| 第二輪 | 「開完會，我想記下一個行動項目」 | 分類判斷（待辦） |
| 第三輪 | 「今天完成了一個重要任務」 | 分類判斷（完成） |

**討論：哪些工作場景最適合用 Skill？**（20 分鐘）
- 學員自己提出他們工作中的應用場景
- 講師協助判斷哪些可以用現有 Skill，哪些需要等待新 Skill

**學員離開時的感受**：「我知道什麼時候該用它，以及怎麼用。」

---

## Stage 4b：個人化調適與回顧（30 分鐘）

**目標**：確認每個人已將 Skill 融入日常工作

**時機**：Stage 4a 結束後間隔 1-2 週

**內容**：

1. 每人分享一個「Skill 幫我省下時間或精力」的例子
2. 收集遇到的問題（Skill 沒有觸發、觸發錯誤、想要的功能沒有）
3. 說明下一步：未來會有更多 Skill，安裝方式相同

---

## 課程成功指標

- 每位學員能在 1 分鐘內完成 Skill 的安裝
- 每位學員在課程結束後的一週內，至少主動使用 Skill 3 次
- 學員能說出「下次遇到 XXX 情境，我會用 Skill 的 XXX 功能」

---

## 講師備注

- Stage 1 最重要，動機不夠強的話，後面的操作會很痛苦
- Stage 2 預留緩衝時間，每個人的環境不同，安裝問題會佔掉很多時間
- Stage 3 結束後讓學員帶著「作業」離開：這週每天用一次 daily-log-coach
- 不要在課程中介紹太多 Skill，一次只學一個，學好再說
