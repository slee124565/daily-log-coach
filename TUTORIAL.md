# Daily Log Coach Skill — 新手體驗課程
## Hands-on Lab

適用對象：已完成 Claude Code 安裝的新手使用者
完成時間：約 40 分鐘
目標：體驗 daily-log-coach skill 的所有核心功能

---

## 課程地圖

```
Lab 1：第一次觸發（10 分鐘）
  └─ 認識 Skill 是什麼、觸發第一筆記錄

Lab 2：五種分類全體驗（15 分鐘）
  ├─ 目標 / 進度 / 完成 / 靈感 / 待辦
  └─ 理解每種分類的自動判斷邏輯

Lab 3：進階功能體驗（10 分鐘）
  ├─ 錯別字自動修正
  ├─ 一次記錄多件事
  └─ 分類模糊時的互動

Lab 4：教練彙整體驗（5 分鐘）
  └─ 觸發 Skill 的累積分析能力
```

---

## 開始之前

**啟動 Claude Code**

打開 Terminal，先切到你的工作目錄再啟動 Claude Code：

```bash
cd /path/to/your/workspace
claude
```

看到 `>` 提示符號，代表已準備好。

**說明**：這堂課的所有練習，都在這個畫面中進行。你只需要輸入文字，Skill 會自動判斷要做什麼。  
`/path/to/your/workspace` 請替換成你實際的專案或工作資料夾路徑。

## Lab 1：第一次觸發（10 分鐘）

### 認識觸發機制

daily-log-coach Skill 的設計方式是「你說話，它自動判斷要不要幫你記錄」。你不需要輸入特殊指令，只要說出包含關鍵詞的句子，Skill 就會啟動。

### 練習 1-1：說一件今天做到的事

在 `>` 提示符號後，輸入以下文字（用你今天真實的事情替換 `XXX`）：

```
我今天完成了 XXX
```

**你應該看到的結果**：
1. Skill 自動判斷這是「完成」類型
2. 記錄寫入今日日誌，包含時間戳和標籤
3. Skill 給你一個回饋問題（例如：「這次完成，哪一個決策讓結果變得更好？」）

**注意觀察**：
- 記錄的格式：`### HH:MM - 完成`
- 標籤：`[標籤: #關鍵詞]`
- 回饋問題在記錄下方出現

### 練習 1-2：查看寫入的日誌

在 Terminal 新開一個分頁，切回同一個 workspace，找到今天的日誌檔案：

```bash
cat daily-logs/$(date +%Y)/$(date +%m)/$(date +%Y-%m-%d).md
```

確認剛才的記錄已經出現在 `## 💭 即時記錄` 區段中。

**這個體驗說明了什麼**：你說了一句話，Skill 做了三件事：判斷類型、寫入檔案、給你回饋。這就是從「打字給 AI 看」到「讓 AI 幫你工作」的差別。

## Lab 2：五種分類全體驗（15 分鐘）

daily-log-coach 有五種分類，每種對應不同的工作情境。這個 Lab 的目標是讓你體驗每一種。

### 分類 1：目標

**適用情境**：設定新的工作目標、方向或計劃

**練習**：輸入以下文字

```
我打算這週把部門的 SOP 文件整理好
```

**觀察重點**：
- 分類應顯示「目標」
- 回饋問題會是關於「這個目標背後想解決的核心問題是什麼？」類型
- 標籤會自動提取你說的關鍵詞（SOP、文件整理 等）

---

### 分類 2：進度

**適用情境**：工作進行中的狀態更新

**練習**：輸入以下文字

```
目前 SOP 文件已經整理了三分之一，預計明天完成
```

**觀察重點**：
- 分類應顯示「進度」
- 注意：Skill 會把這筆記錄和前一筆「目標」分開處理，各自是獨立的條目
- 回饋問題會聚焦在「目前最可能卡關的風險點在哪裡？」類型

---

### 分類 3：完成

**適用情境**：達成目標、結束任務

**練習**：輸入以下文字

```
SOP 文件整理完成了，已經上傳到共用資料夾
```

**觀察重點**：
- 分類應顯示「完成」
- 這是語意最強的分類，當「進度」和「完成」都能匹配時，Skill 優先判斷為「完成」
- 此時你有三筆和 SOP 相關的記錄，請注意是否出現「教練彙整」（見 Lab 4 說明）

---

### 分類 4：靈感

**適用情境**：腦中突然出現的想法、洞見、創意

**練習**：輸入以下文字

```
我在想，如果把常見問題整理成 FAQ，說不定可以減少很多重複的詢問
```

**觀察重點**：
- 分類應顯示「靈感」
- 靈感類型的標籤通常會提取概念性詞彙（FAQ、重複詢問 等）
- 回饋問題會聚焦在「這個想法和你的目標有什麼具體連結？」類型

---

### 分類 5：待辦

**適用情境**：因為看到某個外部資訊（文章、對話、觀察）而產生的行動項目

**練習**：輸入以下文字

```
看到同事分享了一篇關於高效會議的文章，值得研究看看能不能應用
```

**觀察重點**：
- 分類應顯示「待辦」
- 和「靈感」的差別：靈感是你自己想到的，待辦是外部觸發的
- 回饋問題會聚焦在「這個行動項目的優先度如何？」

---

### Lab 2 小結

五種分類的核心邏輯：

| 我說了什麼 | 代表什麼 | Skill 分類 |
|-----------|---------|-----------|
| 我打算、我要... | 還沒開始，方向確定 | 目標 |
| 目前、已經做到... | 進行中 | 進度 |
| 完成了、做好了... | 結束 | 完成 |
| 我在想、突然覺得... | 自己想到的 | 靈感 |
| 看到、讀到、別人說... | 外部觸發 | 待辦 |

## Lab 3：進階功能體驗（10 分鐘）

### 練習 3-1：錯別字自動修正

daily-log-coach 在記錄前會先幫你修正錯別字。

**練習**：輸入一句含有錯字的文字（以下範例故意打錯）

```
今天開會決訂要換用新的排班系統
```

（「決訂」應為「決定」）

**觀察重點**：
- Skill 會先修正錯別字，再進行分類記錄
- 修正幅度小（1-2 字）時直接記錄，不會額外詢問
- 修正幅度大時會先讓你確認修正結果

**延伸練習**：試著輸入一句錯字比較多的句子，看 Skill 如何處理：

```
今天的客供會議很順利，大家都對新方案有共識
```

（「客供」應為「客戶」）— 看看 Skill 是直接修正還是先讓你確認。

---

### 練習 3-2：一次說多件事

有時你想一口氣記錄多件事，Skill 會自動拆分為多筆條目。

**練習**：輸入以下文字

```
今天完成了月報，也收到主管對新企劃的回饋，另外想到可以把報告模板標準化
```

**觀察重點**：
- Skill 會識別出這句話包含三件事：完成（月報）、待辦或進度（主管回饋）、靈感（報告模板）
- 三筆條目使用相同的時間戳
- 每筆條目各自有對應的分類標籤

---

### 練習 3-3：分類模糊時的互動

當 Skill 無法確定分類時，會主動詢問你。

**練習**：輸入一個刻意模糊的句子

```
關於下週的客戶拜訪
```

**觀察重點**：
- 這句話沒有明確的分類訊號
- Skill 應該列出五個分類讓你選擇
- 選擇後繼續完成記錄流程

---

### 練習 3-4：輸入過短的情況

**練習**：輸入一個很短的句子

```
開會
```

**觀察重點**：
- Skill 會詢問「需要補充更多細節嗎？」
- 這是避免記下毫無脈絡的記錄

## Lab 4：教練彙整體驗（5 分鐘）

這是 daily-log-coach 最有深度的功能：當某個主題的記錄累積到 3 筆以上時，Skill 會自動彙整這個主題的發展軌跡，並給出教練式觀察。

### 什麼時候會觸發？

每次記錄後，Skill 會自動搜尋歷史日誌，檢查這次記錄的關鍵詞是否已出現 3 次以上。如果是，就觸發彙整。

### 練習 4-1：追加同主題的記錄

在前面的練習中，你已經記錄了幾筆關於 SOP 文件的內容。現在再加一筆：

```
想到可以用 SOP 文件建立新人教育訓練的素材
```

**如果歷史日誌中 SOP 相關記錄已達 3 筆**，你應該看到：

```
---
**教練彙整｜主題：SOP**

你在這個主題上已累積了 N 筆記錄（最早：YYYY-MM-DD／最近：今天）。

[2-3 句軌跡描述]

教練觀察：[具體、可行動的建議或提問]
---
```

### 如果沒有觸發怎麼辦？

繼續輸入更多相同主題的內容，直到累積 3 筆以上，就會自動觸發。也可以輸入：

```
我在想 SOP 文件的下一步應該怎麼推進
```

### 教練彙整說明了什麼？

一般的筆記工具只會「存檔」。daily-log-coach 在存檔之外，還會主動幫你「看見模式」——你在一個主題上花了多少精力、方向有沒有改變、下一步應該往哪裡走。這是「消費性記錄」和「生產型記錄」的差異。

## 課程總結

完成本課程後，你已經體驗了 daily-log-coach 的所有核心功能：

| 功能 | 對你的工作有什麼用 |
|------|-----------------|
| 五種分類自動判斷 | 不需要思考「這要怎麼分類」，說完就記好了 |
| 即時洞見回饋 | 每次記錄都順便思考一個有意義的問題 |
| 錯別字修正 | 打快一點也沒關係，Skill 會幫你整理 |
| 多筆輸入拆分 | 想到什麼說什麼，不用一件一件分開輸入 |
| 教練彙整 | 不只存檔，還幫你看見自己的成長軌跡 |

---

## 課後練習（帶回家做）

接下來一週，每天用 daily-log-coach 記錄至少 2 件事：
- 早上：記錄今日目標（「我今天要...」）
- 下班前：記錄一件完成的事（「今天完成了...」）

一週後，你會對自己做了什麼、想了什麼，有比以前清楚很多的認識。
